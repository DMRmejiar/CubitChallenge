Preguntas

¿Qué permite la implementación de un interceptor?

Los interceptor permiten evitar la repetición de código en las múltiples llamadas http que se realizan, por ejemplo, en los headers o parámetros que se envían en las request y el manejo de errores de las mismas.

¿Cuándo implementar un guard?

Los guard se utilizan para limitar quien puede y quien no puede acceder a ciertos puntos de la app web, permitiendo proteger ciertas rutas.

¿Qué son una prueba unitaria, una prueba de servicio y una prueba de componente?

Una prueba unitaria, es un test que realiza sobre una función, método o funcionalidad en específico, en ocasiones es necesario realizar múltiples pruebas unitarias sobre una misma función para abarcar todas las posibilidades, una prueba de servicio realiza test a las múltiples funciones de un servicio a diferencia de una pruebas unitaria que solo se enfoca a un único método, adicionalmente una prueba de servicio también puede simular respuestas de peticiones http, y por ultimo una prueba de componente, es un test donde se renderiza un componente y se prueba su funcionalidad, puede probar que si se presiona un botón del componente este realice los cambios o ejecute la función correspondiente a este, y que los cambios se vean reflejados.
